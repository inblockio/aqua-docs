Configure Windsurf's Cascade AI assistant to help you write and maintain documentation. This guide shows how to set up Windsurf specifically for your Mintlify documentation workflow.

Prerequisites

    1. Windsurf editor installed
    1. Access to your documentation repository

Workspace rules

Create workspace rules that provide Windsurf with context about your documentation project and standards. Create .windsurf/rules.md in your project root: ```markdown

Mintlify technical writing rule

Project context

    1. This is a documentation project on the Mintlify platform
    1. We use MDX files with YAML frontmatter
    1. Navigation is configured in docs.json
    1. We follow technical writing best practices

Writing standards

    1. Use second person ("you") for instructions
    1. Write in active voice and present tense
    1. Start procedures with prerequisites
    1. Include expected outcomes for major steps
    1. Use descriptive, keyword-rich headings
    1. Keep sentences concise but informative

Required page structure

Every page must start with frontmatter: `yaml --- title: "Clear, specific title" description: "Concise description for SEO and navigation" --- `

Mintlify components

Callouts

    1. for helpful supplementary information
    1. for important cautions and breaking changes
    1. for best practices and expert advice
    1. for neutral contextual information
    1. for success confirmations

Code examples

    1. When appropriate, include complete, runnable examples
    1. Use for multiple language examples
    1. Specify language tags on all code blocks
    1. Include realistic data, not placeholders
    1. Use and for API docs

Procedures

    1. Use component for sequential instructions
    1. Include verification steps with components when relevant
    1. Break complex procedures into smaller steps

Content organization

    1. Use for platform-specific content
    1. Use for progressive disclosure
    1. Use and for highlighting content
    1. Wrap images in components with descriptive alt text

API documentation requirements

    1. Document all parameters with
    1. Show response structure with
    1. Include both success and error examples
    1. Use for nested object properties
    1. Always include authentication examples

Quality standards

    1. Test all code examples before publishing
    1. Use relative paths for internal links
    1. Include alt text for all images
    1. Ensure proper heading hierarchy (start with h2)
    1. Check existing patterns for consistency
```
