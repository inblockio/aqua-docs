Aquafier is a web-based reference implementation of Aqua Protocol v3, providing a user-friendly interface and HTTP API for digital content signing, provenance verification, and integrity validation. It demonstrates how Aqua Protocol features can be integrated into modern web applications.

Overview

Aquafier combines a Fastify backend with React frontend to deliver a complete solution for document notarization and verification. It's designed as both a production tool and a reference implementation for developers building Aqua-based applications.

Features

    1. Digital Content Signing: Sign documents and data with multiple signature methods
    1. Provenance Verification: Track and verify document history and authenticity
    1. Integrity Validation: Cryptographically verify document integrity
    1. Web Interface: User-friendly React interface for non-technical users
    1. HTTP API: RESTful API for programmatic access
    1. Multi-Party Workflows: Support for documents requiring multiple signatures

Technology Stack

Backend:
    1. Fastify (Node.js web framework)
    1. Prisma ORM
    1. PostgreSQL database
    1. Aqua JavaScript SDK (v3)
Frontend:
    1. React with TypeScript
    1. Chakra UI components
    1. End-to-end testing with Playwright

Hosted Instances

Production: aquafier.inblock.io
    1. Stable production environment
    1. For live document notarization and verification
Development/Testing: dev.inblock.io
    1. Testing environment for new features
    1. Use for development and experimentation

Use Cases

    1. Document Notarization: Notarize legal documents, contracts, and agreements
    1. Certificate Issuance: Issue verifiable certificates and credentials
    1. Multi-Party Signing: Collect signatures from multiple parties
    1. Audit Trail Creation: Build immutable audit trails for compliance
    1. Proof of Concept: Demonstrate Aqua Protocol capabilities
    1. API Integration: Integrate notarization into existing systems

API Access

Aquafier provides a RESTful HTTP API for programmatic access. Developers can integrate notarization and verification into their applications without building infrastructure from scratch. Common API operations:
    1. Create genesis revisions
    1. Add signatures to existing chains
    1. Witness chains on blockchain
    1. Verify chain integrity
    1. Retrieve chain history

Development

Aquafier-JS is open source and can be self-hosted for custom deployments.

Requirements

    1. Docker and Docker Compose
    1. PostgreSQL database
    1. Node.js environment

Deployment

Deploy using Docker Compose: ``bash

Prepare environment file

cp deployment/.env.sample .env

Start services

docker compose -f deployment/docker-compose-prod.yml up -d ` Deployment Options:
    1. Local: No proxy, exposed ports for development
    1. Dev: With Let's Encrypt SSL and DNS
    1. Prod: Production configuration with SSL

Configuration

Key environment variables: `bash

Database

DATABASE_URL=postgres://user:password@host:port/database

Server

HOST=0.0.0.0 PORT=3000

Frontend/Backend URLs (for proxy)

FRONTEND_URL=https://your-domain.com BACKEND_URL=https://api.your-domain.com

Twilio (optional, for SMS verification)

TWILIO_ACCOUNT_SID=your_sid TWILIO_AUTH_TOKEN=your_token ``

Architecture

Aquafier's modular architecture makes it suitable as a foundation for custom applications:
    1. API Layer: RESTful endpoints for all Aqua operations
    1. Service Layer: Business logic for chain management
    1. Data Layer: Prisma ORM with PostgreSQL
    1. Storage: File system or S3-compatible storage
    1. Backup: Automated backup with configurable retention
